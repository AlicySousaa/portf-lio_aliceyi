Organização do código

A retrospectiva de cada sprint foi essencial para ajustar o escopo. Por exemplo, na Sprint 3 percebi que o NativeWind estava causando problemas no Windows e tomei a decisão de migrar para StyleSheet nativo — uma escolha técnica que veio de uma retrospectiva honesta.

💻

decisões técnicas

POR QUE

React Native

Já tinha familiaridade com React e queria uma solução cross-platform real. O ecossistema Expo simplificou muito o setup inicial e permitiu focar na lógica do app.

POR QUE

Firebase

Backend as a Service que eliminou a necessidade de um servidor próprio. Autenticação, Firestore em tempo real e regras de segurança por usuário — tudo integrado, perfeito para o escopo acadêmico.

POR QUE

TypeScript

Tipagem estática que previne bugs silenciosos. Em um projeto solo, o TypeScript funciona como um par que revisa o código em tempo real — interfaces claras para InventoryItem, RoutineItem e GoalItem.

DECISÃO DE ARQUITETURA

StyleSheet nativo

Tentei NativeWind mas havia incompatibilidade no Windows. Optei por StyleSheet puro do React Native — mais verboso, porém 100% confiável e sem dependências extras.

Organização do código

Separei o projeto em hooks, services e components para isolar responsabilidades. O hook useMarki centraliza os listeners do Firestore, evitando memory leaks com cleanup correto. O firestoreActions valida dados antes de salvar — preço negativo não passa, por exemplo.

⚡

desafios & aprendizados

✓

Memory leak nos listeners do Firestore

Os listeners onSnapshot ficavam abertos mesmo após logout. Resolvi com cleanup correto no useEffect — variáveis declaradas fora do callback do onAuthStateChanged e limpas no return do effect.

✓

NativeWind incompatível no Windows

O metro.config.js do NativeWind falhava por questão de paths no Windows. Tomei a decisão de usar StyleSheet nativo — mais trabalho, zero dependência problemática.

✓

Teclado cobrindo o modal no celular

No iOS, o teclado cobria os campos de input do modal. Solução: KeyboardAvoidingView com behavior='padding' no iOS e 'height' no Android.

✓

API Key exposta no código

As credenciais do Firebase estavam hardcoded. Migrei para variáveis de ambiente com EXPO_PUBLIC_ e configurei o .gitignore corretamente.
